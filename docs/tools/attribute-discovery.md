# Attribute-driven discovery

Convention-over-configuration registration of commands, routes, and view
composers via PHP attributes.

`Package::discoversWithAttributes()` scans a package's `src/` for classes
carrying discovery attributes and wires them through the fluent API, with
no explicit `hasCommand()` / `hasViewComposer()` calls required.

```php
$package->discoversWithAttributes(?string $directory = null, ?string $namespace = null);
```

- `directory` — filesystem directory to scan; defaults to the package's
  `src/` (`packageBasePath('src')`).
- `namespace` — PSR-4 root for that directory; defaults to the package's
  autoload-detected namespace.

Calling the method only sets a flag and stores the overrides; discovery
itself runs at `packageBooted()` time. The trait
`Simtabi\Laranail\PackageTools\Concerns\Package\DiscoversWithAttributes`
exposes `isUsingAttributeDiscovery()`, `getAttributeDiscoveryDirectory()`,
`getAttributeDiscoveryNamespace()`, `getDiscoveredRoutes()`, and
`getDiscoveredViewComposers()` for inspection.

`runAttributeDiscovery()` performs the scan and returns per-type counts
`{commands, routes, view_composers}`.

## What is wired

| Attribute | Wired via |
|---|---|
| `#[AsArtisanCommand]` | `hasCommand()` |
| `#[AsRoute]` | a route descriptor registry the provider iterates at boot |
| `#[AsViewComposer]` | `hasViewComposer()` (one per view) |

`#[AsFacade]` is **not** processed by `discoversWithAttributes()`; facade
classes are generated by the `laranail::package-tools.ide-helper` command — see
[ide-helper.md](ide-helper.md).

## The attributes

All four live under
`Simtabi\Laranail\PackageTools\Attributes\`.

### `#[AsArtisanCommand]`

`#[Attribute(Attribute::TARGET_CLASS)]`

```php
public function __construct(
    public string $signature,
    public ?string $description = null,
)
```

```php
#[AsArtisanCommand(signature: 'foo:run', description: 'Run the foo task')]
final class FooRunCommand extends Command { /* … */ }
```

### `#[AsRoute]`

`#[Attribute(Attribute::TARGET_CLASS | Attribute::IS_REPEATABLE)]` —
repeatable, so one controller can declare multiple routes.

```php
public function __construct(
    public string $method,
    public string $uri,
    public ?string $name = null,
    /** @var string[] */
    public array $middleware = [],
)
```

```php
#[AsRoute(method: 'GET',  uri: '/foo')]
#[AsRoute(method: 'POST', uri: '/foo', name: 'foo.create', middleware: ['web'])]
final class FooController { public function __invoke() { /* … */ } }
```

### `#[AsViewComposer]`

`#[Attribute(Attribute::TARGET_CLASS)]`

```php
public function __construct(
    /** @var string|string[] */
    public string|array $views,
)
```

```php
#[AsViewComposer(views: ['layouts.app', 'partials.header'])]
final class AppViewComposer { public function compose(View $view) { /* … */ } }

// A single view string is also accepted:
#[AsViewComposer(views: 'layouts.app')]
```

### `#[AsFacade]`

`#[Attribute(Attribute::TARGET_CLASS)]`. Constructor
`(string $alias, ?string $accessor = null)`. Consumed by the
`laranail::package-tools.ide-helper` command, not by `discoversWithAttributes()`. See
[ide-helper.md](ide-helper.md).

## The discoverer

`Simtabi\Laranail\PackageTools\Services\Discovery\AttributeDiscoverer`
backs discovery. It is stdlib-only (no third-party auto-discoverer
dependency): it recursively walks `*.php` files, derives each FQCN from
the file path under the PSR-4 root, reflects the class, and yields
`{class: ReflectionClass, attributes: list<ReflectionAttribute>}` for
every class carrying the requested attribute (matched
`IS_INSTANCEOF`). A non-existent directory throws a `RuntimeException`;
files that fail to reflect are skipped.
